BlackTelperion.multiprocessing module

Generic tools for distributing computationally intensive tasks across multiple threads.

BlackTelperion.multiprocessing.parallel_chunks(function, data, *args, **kwds)[source]

Run a function that operates per-point or per-pixel on smaller chunks of a point cloud or image dataset in parallel. Only use for expensive operations as otherwise overheads (writing files to cache, spawning threads, loading files from cache) are too costly.

Parameters:

  • function – the function to run on each chunk of the dataset. Must take a BlackCloud or BlackImage dataset as it’s first argument and also return a BlackCloud or BlackImage dataset (cf., mwl(…), get_hull_corrected(…)).

  • data (BlackCloud or BlackImage) – data to run the function on.

  • args (tuple) – tuple of arguments to pass to the function.

  • nthreads (int) – the number of threads to spawn. Default is the number of cores - 2. Negative numbers will be subtracted from the number of cores.

  • **kwds – any other keywords are passed to the function

Returns:

Nothing

BlackTelperion.multiprocessing.parallel_datasets(function, in_paths, out_paths=None, nthreads=-2, **kwds)[source]

Parallelise a single function across many BlackData datasets.

Parameters:

  • function – the function to run on each dataset. This should take an input path (string) as its first input and output path (also string) as its second output. Anything returned by the function will be ignored.

  • in_paths (list) – a list of input paths, each of which will be passed to function in each thread.

  • out_paths (list) – a list of corresponding output paths that each function should write to. Defaults to in_paths.

  • nthreads (int) – the number of threads to spawn. Default is the number of cores - 2. Negative numbers are subtracted from the total number of cores.

  • **kwds – any keywords are passed directly to function in each thread.

Returns:

Nothing
